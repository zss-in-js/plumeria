The `pseudo` utility in the `@plumeria/core` package provides a powerful way to apply CSS pseudo-classes and pseudo-elements programmatically. You can use these utilities as selectors within an array or combine multiple pseudo states using the `cx` function. This reference covers all available pseudo-classes, pseudo-elements, and functional pseudo selectors, with detailed descriptions and usage examples.

## Overview

The `pseudo` utility allows developers to apply CSS pseudo-classes and pseudo-elements directly within JavaScript or TypeScript code using `@plumeria/core`. This is particularly useful for dynamic styling in modern web applications, such as single-page apps or component-based frameworks. By leveraging these utilities, you can create responsive and interactive styles with minimal CSS boilerplate.

## Pseudo-Classes

### Standard Pseudo-Classes

Pseudo-classes target elements based on their state, position, or user interaction. Below is a comprehensive list of all supported pseudo-classes in the `pseudo` utility.

| Pseudo-Class                  | Description                                                                  | Example Use Case                                      |
| ----------------------------- | ---------------------------------------------------------------------------- | ----------------------------------------------------- |
| `css.pseudo.active`           | Styles an element when it is being activated (e.g., clicked).                | Highlight a button during a click event.              |
| `css.pseudo.anyLink`          | Targets any link, whether visited or unvisited.                              | Style all anchor tags uniformly.                      |
| `css.pseudo.autoFill`         | Targets form elements auto-filled by the browser.                            | Customize the appearance of auto-filled input fields. |
| `css.pseudo.buffering`        | Styles media elements (e.g., `<video>`) while buffering.                     | Display a loading state for a video player.           |
| `css.pseudo.checked`          | Targets checked form elements (e.g., checkboxes, radio buttons).             | Style checked radio buttons or checkboxes.            |
| `css.pseudo.default`          | Targets the default element among a group (e.g., default button in a form).  | Highlight the default submit button.                  |
| `css.pseudo.defined`          | Targets elements that are defined (e.g., custom elements with a definition). | Style custom web components after definition.         |
| `css.pseudo.disabled`         | Targets disabled form elements or controls.                                  | Gray out disabled buttons or inputs.                  |
| `css.pseudo.empty`            | Targets elements with no children or text content.                           | Hide empty `<div>` elements.                          |
| `css.pseudo.enabled`          | Targets enabled form elements or controls.                                   | Style enabled inputs differently from disabled ones.  |
| `css.pseudo.first`            | Targets the first element in a group (e.g., in a `:where()` context).        | Style the first item in a dynamic list.               |

| `css.pseudo.firstOfType`      | Targets the first sibling of its type.                                       | Style the first `<h1>` among siblings.                |

| `css.pseudo.focusWithin`      | Targets an element containing a focused descendant.                          | Style a form when any input inside is focused.        |
| `css.pseudo.fullscreen`       | Targets an element in fullscreen mode.                                       | Style a video player in fullscreen.                   |

| `css.pseudo.inRange`          | Targets form elements with values within a valid range.                      | Highlight valid range inputs.                         |
| `css.pseudo.indeterminate`    | Targets checkboxes or radio buttons in an indeterminate state.               | Style indeterminate checkboxes.                       |
| `css.pseudo.invalid`          | Targets form elements with invalid input.                                    | Highlight invalid input fields.                       |

| `css.pseudo.lastOfType`       | Targets the last sibling of its type.                                        | Style the last `<p>` among siblings.                  |
| `css.pseudo.left`             | Targets elements in a paged context (e.g., print layouts).                   | Style left-hand pages in a print stylesheet.          |
| `css.pseudo.link`             | Targets unvisited links.                                                     | Style unvisited `<a>` tags.                           |
| `css.pseudo.modal`            | Targets elements in a modal state (e.g., dialogs).                           | Style modal dialogs.                                  |
| `css.pseudo.muted`            | Targets media elements that are muted.                                       | Style muted video players.                            |
| `css.pseudo.onlyChild`        | Targets an element that is the only child of its parent.                     | Style a lone `<span>` in a `<div>`.                   |
| `css.pseudo.onlyOfType`       | Targets an element that is the only one of its type among siblings.          | Style a single `<h2>` among siblings.                 |

## Pseudo-Elements

Pseudo-elements allow styling specific parts of an element, such as its first letter or content inserted before/after it.

| Pseudo-Element                   | Description                                              | Example Use Case                                 |
| -------------------------------- | -------------------------------------------------------- | ------------------------------------------------ |
| `css.pseudo.after`               | Styles content inserted after an element.                | Add an icon after a link.                        |
| `css.pseudo.backdrop`            | Targets the backdrop of a dialog or fullscreen element.  | Style the dimmed background of a modal.          |
| `css.pseudo.before`              | Styles content inserted before an element.               | Add a bullet before a list item.                 |
| `css.pseudo.cue`                 | Styles WebVTT cues for media elements (e.g., subtitles). | Customize subtitle appearance in videos.         |
| `css.pseudo.detailsContent`      | Targets the content of a `<details>` element.            | Style the content inside an accordion.           |
| `css.pseudo.firstSelectorButton` | Targets the first selector button in a UI component.     | Style the first button in a custom form control. |
| `css.pseudo.firstLetter`         | Styles the first letter of an element’s text.            | Enlarge the first letter of a paragraph.         |
| `css.pseudo.firstLine`           | Styles the first line of an element’s text.              | Bold the first line of a paragraph.              |
| `css.pseudo.grammarError`        | Styles text marked as a grammar error by the browser.    | Highlight grammar errors in a text editor.       |
| `css.pseudo.marker`              | Styles list item markers (e.g., bullets or numbers).     | Customize bullet styles in a list.               |
| `css.pseudo.placeholder`         | Styles the placeholder text in inputs or textareas.      | Change placeholder text color.                   |
| `css.pseudo.selection`           | Styles selected text within an element.                  | Customize text selection highlight.              |
| `css.pseudo.spellingError`       | Styles text marked as a spelling error by the browser.   | Highlight spelling errors in a text editor.      |
| `css.pseudo.targetText`          | Styles text targeted by a URL fragment.                  | Highlight text linked via a URL anchor.          |
| `css.pseudo.viewTransition`      | Styles elements during a view transition animation.      | Style elements during page transitions.          |

## Functional Pseudo-Classes

Functional pseudo-classes accept arguments to define complex selectors dynamically.

| Functional Pseudo-Class                | Description                                                               | Example Use Case                                       |
| -------------------------------------- | ------------------------------------------------------------------------- | ------------------------------------------------------ |
| `css.pseudo.fn.dir(direction)`         | Targets elements based on text direction (e.g., `ltr` or `rtl`).          | Style text differently for RTL languages.              |
| `css.pseudo.fn.has(selector)`          | Matches elements containing specific descendants.                         | Style a `<div>` containing a specific `<span>`.        |
| `css.pseudo.fn.host(selector)`         | Targets the shadow DOM host matching the selector.                        | Style a shadow DOM host based on a class.              |
| `css.pseudo.fn.hostContext(selector)`  | Matches the shadow DOM host based on its context.                         | Style a host when a parent has a specific class.       |
| `css.pseudo.fn.is(selector)`           | Matches elements that match any of the provided selectors.                | Combine multiple selectors for flexible styling.       |
| `css.pseudo.fn.lang(language)`         | Targets elements with a specific language (e.g., `en`, `ja`).             | Style text for a specific language.                    |
| `css.pseudo.fn.nthChild(formula)`      | Targets elements based on their position among siblings (e.g., `2n+1`).   | Style odd-numbered list items.                         |
| `css.pseudo.fn.nthLastChild(formula)`  | Targets elements based on their position from the last sibling.           | Style the second-to-last list item.                    |
| `css.pseudo.fn.nthLastOfType(formula)` | Targets elements of a type based on their position from the last sibling. | Style the last `<p>` among siblings.                   |
| `css.pseudo.fn.nthOfType(formula)`     | Targets elements of a type based on their position among siblings.        | Style every third `<div>` of its type.                 |
| `css.pseudo.fn.not(selector)`          | Targets elements that do not match the provided selector.                 | Style all elements except those with a specific class. |
| `css.pseudo.fn.state(state)`           | Targets elements in a specific custom state.                              | Style elements with a custom state attribute.          |
| `css.pseudo.fn.where(selector)`        | Matches elements that match the provided selector with specificity.       | Style elements matching complex conditions.            |

## Usage Examples

Below are practical examples demonstrating how to use the `pseudo` utility with `@plumeria/core`.

### Example 1: Styling a Button with Hover and Focus

```ts title="TypeScript Example"
import { css } from '@plumeria/core';

const buttonStyles = css.create({
  button: {
    background: 'blue',
    color: 'white',
    padding: '10px 20px',
    [css.pseudo.hover]: {
      background: 'darkblue',
    },
    [css.pseudo.focus]: {
      outline: '2px solid yellow',
    },
  },
});
```

## Example 2: Combining Pseudo-Classes with cx

```ts
import { cx } from '@plumeria/core';

const combinedStyles = cx(css.pseudo.active, css.pseudo.focusVisible, css.pseudo.fn.has('div'));
```

## Example 3: Using Functional Pseudo-Classes

```ts
import { css } from '@plumeria/core';

const listStyles = css.create({
  list: {
    [css.pseudo.fn.nthChild('2n')]: {
      background: 'lightgray',
    },
    [css.pseudo.fn.has('img')]: {
      border: '1px solid black',
    },
  },
});
```

## Example 4: Styling Pseudo-Elements

```ts
import { css } from '@plumeria/core';

const textStyles = css.create({
  text: {
    [css.pseudo.firstLetter]: {
      fontSize: '1.5em',
      color: 'red',
    },
    [css.pseudo.before]: {
      content: '"★ "',
      color: 'gold',
    },
  },
});
```
