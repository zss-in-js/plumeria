## 7.0.0

- **Add eslint-plugin no-combinator rule**
- **Add marker extended API**

## Changes

- "&" is not allowed.
- selector combinator is not allowed.
- Selectors starting with "[" are now allowed.

### Syntax forbidden to maintain atomicity
I get an error due to eslint rules.
```ts
":hover > div": {...}
```

## new API
`marker` and `extended`.
<MarkerExtended />

```tsx
import * as css from '@plumeria/core';

const styles = css.create({
  table: {
    ...css.marker('table', ':hover'), // [!code ++]
    width: '100%',
    height: 200,
    listStyleType: 'none',
  },
  list: {
    transition: 'all 0.5s',
    [css.extended('table', ':hover')]: { // [!code ++]
      color: 'green',
      scale: 1.2,
      ':hover': {
        color: 'pink',
        scale: 1.5,
      },
    },
  },
});;

const Test = () => {
  return (
    <div>
      <ul className={css.props(styles.table)}>
        <li className={css.props(styles.list)}>Tests</li>
      </ul>
    </div>
  );
};
```

## Migrate guide
- Replace `"&"` selector with `":"` or `"["`
- Replace combinator syntax with `marker` `extended`.
