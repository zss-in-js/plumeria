# Release Notes

## Breaking Change
css.props() renamed to css.use()

**Before (8.x)**
```tsx
import css from '@plumeria/core';

<div className={css.props(styles.featureCard)} />
```
**After (9.0.0)**
```tsx
import css from '@plumeria/core';

<div className={css.use(styles.featureCard)} />
```

## Reason for Change
- The name "props" did not accurately represent its actual meaning (conversion to a class name).
- css.use() now more intuitively and concisely conveys the intent of "use this style."

## Migration
Simply replace `css.props(` with `css.use(`.

## 8.x Release Notes

- 8.0.1 fixes some edge-case bugs in development mode, such as styles being lost when opening in a new tab.
- 8.0.2 fixes first line comment out parsing error.
- 8.0.3 fixes fs.globSync was slow and has been replaced with @rust-gear/glob, which is tuned for speed.
