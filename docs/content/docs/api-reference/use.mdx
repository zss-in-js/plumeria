---
title: 'use'
description: 'API reference for the use()'
---

```ts
export type use = (...rules: (false | CSSProperties | null | undefined)[])=> string;
export const use: use;
```

The `use()` returns style object as hashed classes together while **filtering out falsy values** like **undefined**, **false**, or **null**. It is useful when conditionally combining styles.

## Example
```ts title="TypeScript"
import * as style from '@plumeria/core';

const styles = style.create({
  one: {
    display: 'flex',
    justifyContent: 'space-between',
  },
  two: {
    fontSize: '14px',
    color: '#606060',
  },
});
```

```jsx
<div className={style.use(styles.one, styles.two)}> // [!code ++]
  {/* ... */}
</div>
```

## Usage Patterns

### Conditional styles

Only truthy values are preserved.

```ts
const isActive = true;

const className = style.use(
  styles.one,
  isActive && styles.two
);
```

### With ternary

```ts
const className = style.use(
  isDark ? styles.dark : styles.light,
  isHovered && styles.hovered
);
```

---

## Return value
Returns a hashed `string` of space-separated styleObject, ignoring falsy inputs:

```ts
style.use(styles.one, false, undefined, styles.two);
```

<Callout title="Good to know">
  use(), like all APIs, have no runtime. They are controlled at build time by plugins, so functions themselves are replaced by hashes.
</Callout>
