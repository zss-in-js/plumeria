---
title: css.createTheme
description: 'API reference for the createTheme'
---

```ts
export type createTheme = <const T extends CreateTheme>(rule: T)=> ReturnVariableType<T>;
export const createTheme: createTheme;
```
`createTheme` defines global CSS variables for each theme.  
Since a unique hash is automatically prepended to the generated variable names, there is no risk of name collisions with other themes or global CSS variables.

## Example

```ts
import * as css from '@plumeria/core'

export const theme = css.createTheme({
  colors: {
    default: 'black',
    dark: 'white'
  },
  bg: {
    default: 'white',
    dark: 'black'
  }
})
```

This will generate the following CSS

```css
:root {
  --xukqmak7-colors: black;
  --xukqmak7-bg: white;
}

[data-theme="dark"] {
  --xukqmak7-colors: white;
  --xukqmak7-bg: black;
}
```

### Using Themes

To use themes, you need to add the `data-theme` attribute to the root element.

```tsx
<html data-theme="dark">
  <body>
    ...
  </body>
</html>
```



## Media Queries

`createTheme` can also be used with media queries.

```ts
export const theme = css.createTheme({
  colors: {
    default: 'black',
    dark: 'white',
    '@media (prefers-color-scheme: dark)': 'white'
  }
})

// theme evaluates to:
// {
//   colors: 'var(--xukqmak7-colors)'
// }
```

This will generate the following CSS

```css
:root {
  --xukqmak7-colors: black;
}

[data-theme="dark"] {
  --xukqmak7-colors: white;
}

@media (prefers-color-scheme: dark) {
  :root {
    --xukqmak7-colors: white;
  }
}
```


<Callout title="Good to know">
All APIs, including createTheme, are not compiled until the variable is used.
</Callout >
